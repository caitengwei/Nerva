# Nerva 设计与测试文档（中文）

更新时间：2026-03-03

口径说明：本目录以当前仓库代码为主口径，来源于 `src/nerva/`、`tests/`、`scripts/bench/` 与 `README.md`。如果线上行为与本文不一致，以实际运行结果和最新回归为准。

## 文档定位

这套文档不是“接口清单”，而是面向协作开发的工程文档。目标是让新同学在半天内建立正确心智模型，并让老同学在改动时能快速定位影响面、测试面和性能面。

文档覆盖四个维度：
- 架构设计：系统边界、数据流、并发模型、故障处理。
- 功能设计：按模块说明设计意图、关键 API、扩展方式。
- 本地测试：如何稳定复现、如何补测试、如何做回归保护。
- 性能测试：如何跑基准、如何读结果、如何新增性能场景。

## 推荐阅读路径

### 路径 A：新同学上手

1. 先看《架构设计》，建立“请求如何穿过系统”的主链路认识。
2. 再看《功能设计》，把主链路映射到具体模块和关键 API。
3. 最后看《本地测试指南》，确保能独立验证改动。

### 路径 B：准备做功能改动

1. 从《功能设计》里找到目标模块及其边界。
2. 回到《架构设计》确认跨模块影响。
3. 按《本地测试指南》补单测/集成回归。

### 路径 C：准备做性能优化

1. 先读《架构设计》的关键路径与并发模型。
2. 再读《性能测试指南》的执行口径和指标解释。
3. 优化后按统一口径复跑并对比基线。

## 协作建议

- 文档和代码要同频更新：行为变更、接口变更、质量数据变化都应同步回填。
- 文档优先写“为什么这样做”，再写“怎么做”。
- 对关键判断给出可复现证据（代码路径、测试命令、结果目录）。

## 当前适用范围

- 服务入口：`build_nerva_app(...)`、`serve(...)`。
- 默认端点：`POST /rpc/{pipeline_name}`、`GET /v1/health`、`GET /v1/models`、`GET /metrics`。
- 测试体系：`pytest` + `pytest-asyncio`，`ruff`，`mypy`。
- 性能体系：Phase 2 基准用例 + Phase 7 对照压测工具链。
