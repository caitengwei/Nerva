# Nerva 设计与测试文档（中文）

更新时间：2026-03-03

这组文档给两类人用：
- 新加入项目的同学，用来快速建立系统全貌。
- 正在改代码的同学，用来确认影响面、回归范围和性能口径。

内容来自当前仓库代码（`src/nerva/`、`tests/`、`scripts/bench/`）和 `README.md`。如果文档描述与实际运行不一致，以代码和最新回归结果为准。

## 建议阅读顺序

### 第一次接触 Nerva

1. 先看《架构设计》，搞清请求是怎么穿过系统的。
2. 再看《功能设计》，把主链路落到具体模块和 API。
3. 最后看《本地测试指南》，确保你能在本地稳定验证改动。

### 准备改功能

1. 在《功能设计》里先定位模块边界。
2. 回到《架构设计》确认跨模块影响。
3. 按《本地测试指南》选最小回归组合。

### 准备做性能优化

1. 先看《架构设计》里的关键路径和并发模型。
2. 再看《性能测试指南》里的执行口径和指标解释。
3. 复跑统一矩阵，再跟历史结果对比。

## 协作约定

- 功能行为有变化时，文档要跟着改，不要只改代码。
- 文档里尽量写清“为什么这么做”，不只写“怎么做”。
- 涉及判断的结论，尽量给可复现证据（代码路径、命令、结果目录）。
