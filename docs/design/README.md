## 推荐阅读顺序

1. 先读架构设计，建立全局心智模型。
2. 再读功能设计，定位具体模块与 API。
3. 开发前读本地测试指南，按分层补充单测/集成测试。
4. 做性能优化或对比实验时读性能测试指南。

## 适用范围

- 代码基线：`main` 分支已完成 Phase 0 ~ Phase 5；Phase 7 benchmark 工具链已落地。
- 服务入口：`build_nerva_app(...)` 与 `serve(...)`。
- 默认对外端点：`POST /rpc/{pipeline_name}`、`GET /v1/health`、`GET /v1/models`、`GET /metrics`。
